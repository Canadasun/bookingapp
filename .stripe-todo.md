# Stripe price ID update required

When prices changed (Basic $19, Pro $39, Unlimited $79), you MUST update the
Stripe Price IDs in your environment config and in any webhook handlers that
reference the old price IDs.

Checklist:
- [ ] Create new Price objects in Stripe Dashboard for each plan
- [ ] Update STRIPE_PRICE_BASIC, STRIPE_PRICE_PRO, STRIPE_PRICE_UNLIMITED env vars
      in Railway (production) and .env.local (dev)
- [ ] If you have a checkout session handler that maps price_id → plan tier,
      update that mapping too
- [ ] Archive (do NOT delete) the old Price objects in Stripe so existing
      subscriptions don't break — Stripe lets you archive prices
- [ ] Test a new checkout flow end-to-end in Stripe test mode before going live
- [ ] If you have any active paying subscribers, grandfather them at old price
      or communicate the change before migrating
